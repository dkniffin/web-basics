HTML attributes

Each HTML tag has a few parts to it, such as the opening tag, the closing tag and attributes.

This exercise focuses on the HTML attributes. A list of some of the most common attributes can be found here. The beginning of this document will go through a few of the most commonly used global attributes that can be used on any HTML element. After that, there will be some examples of specific attributes, especially focused on forms and input elements.

Common HTML Attributes

id

The first important attribute is id, which is used to define a unique identifier within the whole document. It's purpose is to identify the element when linking, styling, or scripting:

  • For linking, you can create a URL that links directly to any element on a page by adding a "#identifier-name" to the end. For example, clicking on a link with an href of https://example.com#foo-bar will navigate the user to https://example.com, with the page scrolled down to focus on an element with id="foo-bar".
  • For styling, in CSS you can target an individual element on the page using the id. For example, the CSS #id { color: red; } would make the text color red for the element with id="foo-bar".
  • For scripting, in Javascript you can get a reference to an element using the id. For example, to get a reference to the element with id="foo-bar", you can use the JS code document.getElementById("foo-bar") or document.querySelector("#foo-bar").

class

The next important attribute is class. This attribute defines a space-separated list of classes for an element. For example, if an element has class="foo bar", then that element has the class "foo" and the class "bar". These classes are primarily used for CSS styling, though they can be used for scripting as well. In the above example, that element would have all of the CSS styles for both .foo and .bar.

data-* attributes

The final set of common HTML attributes that we'll cover here are ones that start with the data- prefix. These attributes are the way for developers to define their own data about an element, and they are typically used to pass data to Javascript. We'll cover the in more depth when we learn JS.

Side-note: the * in the title for this section is called a wild-card. You'll see it occasionally in the context of programming. It basically means "anything could go here". For example, this section is referring to data-foo, data-bar, data-something-else, data-whatever-custom-thing-you-can-think-of, etc.
